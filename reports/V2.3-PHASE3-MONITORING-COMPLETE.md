## 功能实现详情

### 1. Prometheus 指标采集

**HTTP 指标:**
- `http_requests_total` - HTTP 请求总数（按 method/endpoint/status 标签）
- `http_request_duration_seconds` - HTTP 请求延迟直方图

**数据库指标:**
- `db_connections_active` - 活跃数据库连接数
- `db_query_duration_seconds` - 数据库查询延迟直方图

**业务指标:**
- `products_scanned_total` - 扫描商品总数
- `history_records_total` - 历史记录总数

**特性:**
- ✅ 支持 Prometheus 原生格式导出
- ✅ 提供 in-memory 降级方案（当 prometheus_client 未安装时）
- ✅ 单例模式，线程安全
- ✅ 监控开销 <1ms

### 2. 请求追踪

**核心功能:**
- ✅ 请求链路追踪（trace_id 生成和管理）
- ✅ Span 记录（支持多级 span）
- ✅ 慢请求检测（可配置阈值）

**API:**
```python
# 手动追踪
trace_id = start_trace(endpoint='/api/products')
add_span(trace_id, 'db_query', duration=0.05)
end_trace(trace_id)

# 上下文管理器
with trace_request('/api/products') as ctx:
    # 业务逻辑
    pass

# 获取追踪数据
trace = get_trace(trace_id)
slow_traces = get_slow_traces(threshold_ms=1000)
```

### 3. 错误追踪

**核心功能:**
- ✅ 错误捕获（自动记录堆栈跟踪）
- ✅ 错误统计（按类型、端点分组）
- ✅ 错误详情（完整上下文信息）
- ✅ 错误聚合（相似错误自动分组）

**API:**
```python
# 手动追踪
track_error(exception, context={'user_id': 'xxx'}, endpoint='/api/test')

# 上下文管理器
with track_errors(endpoint='/api/test'):
    # 业务逻辑
    pass

# 查询错误
errors = get_errors(hours=24)
top_errors = get_top_errors(limit=10)
error_rate = get_error_rate()
```

### 4. 慢查询日志

**核心功能:**
- ✅ 数据库慢查询检测（默认阈值 1 秒）
- ✅ 查询归一化（相似查询自动分组）
- ✅ 查询分析（统计信息）
- ✅ 可配置阈值

**API:**
```python
# 记录慢查询
log_slow_query(query, duration=2.0, params={'id': 1})

# 上下文管理器
with time_query("SELECT * FROM products") as timer:
    # 执行查询
    pass

# 查询慢查询
queries = get_slow_queries(limit=100)
stats = get_query_monitor().get_query_stats()
```

## API 端点

所有端点均在 `/api/v2/monitor` 前缀下：

| 端点 | 方法 | 说明 |
|------|------|------|
| `/health` | GET | 健康检查 |
| `/metrics/prometheus` | GET | Prometheus 格式指标 |
| `/metrics/summary` | GET | JSON 格式指标摘要 |
| `/traces` | GET | 请求追踪数据 |
| `/slow-queries` | GET | 慢查询日志 |
| `/errors` | GET | 错误追踪数据 |
| `/stats` | GET | 综合监控统计 |

### 端点参数示例

**获取慢追踪:**
```
GET /api/v2/monitor/traces?slow=true&threshold=500&limit=50
```

**获取错误按类型分组:**
```
GET /api/v2/monitor/errors?by_type=true&hours=24
```

**获取慢查询统计:**
```
GET /api/v2/monitor/slow-queries?stats=true&limit=100
```

## 使用示例

### 在 Flask 应用中集成

```python
from flask import Flask
from web.routes.monitor import init_monitor_routes
from src.monitoring import time_request, track_error

app = Flask(__name__)
init_monitor_routes(app)

@app.route('/api/products')
def get_products():
    with time_request('GET', '/api/products') as timer:
        try:
            # 业务逻辑
            products = get_all_products()
            timer.status = 200
            return jsonify(products)
        except Exception as e:
            timer.status = 500
            track_error(e, endpoint='/api/products')
            raise
```

### 在数据库操作中集成

```python
from src.db.monitor import time_query

def get_product_by_id(product_id: int):
    query = "SELECT * FROM products WHERE id = ?"
    with time_query(query, params={'id': product_id}):
        return db.execute(query, (product_id,))
```

### 在业务逻辑中记录指标

```python
from src.monitoring import get_prometheus_metrics

def scan_products():
    metrics = get_prometheus_metrics()
    
    for product in products:
        # 扫描逻辑
        process_product(product)
        
        # 记录指标
        metrics.record_product_scanned(1)
```
